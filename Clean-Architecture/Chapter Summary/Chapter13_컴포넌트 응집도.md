# Chapter 13 컴포넌트 응집도

(엄격하게 제약을 가하는 역할)
CCP : 공통 폐쇄 원칙
CRP : 공통 재사용 원칙

---

REP : 중요한 테마나 목적으로 묶여있어야 한다.
(관리 및 개발 시점에서 너무도 당연한 이야기)

CCP

- 동일한 이유와 시점 (SRP시점) -> 유지보수성 측면
- 변경에 닫혀있는 클래스를 하나의 컴포넌트로 묶음
  > "컴포넌트 수준의 SRP다."

CRP

- 어떤 클래스를 한 데 묶어도 되는지? 보다는 어떤 클래스를 한데 묶으면 안되는지
  CRP ⊃ ISP
